Class: GraphQL::Schema::Enum::UnresolvedValueError < GraphQL::Error This is raised when either:

  • A resolver returns a value which doesn’t match any of the enum’s configured values; - Or, the resolver returns a value which matches a value, but that value’s authorized? check returns false. In either case, the field should be modified so that the invalid value isn’t returned. GraphQL::Schema::Enum subclasses get their own subclass of this error, so that bug trackers can better show where they came from. Instance methods: initialize
